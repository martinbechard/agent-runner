Read `plugins/methodology-runner-skills/AUTHORING-CONTEXT.md` first.
It has before/after examples for each lesson. Key patterns:

- Hybrid structure (procedure + transforms) > either alone
- Schema shape drives behavior > prose instructions
- Rules fighting agent defaults need: rule + example + counter-example
- Quote Test + Invention Traps suppress all phantoms. Always include.
- No flowcharts as primary structure — they create blind spots
- Test with novel sources, not the skill's own examples
- Phase skills defer traceability to traceability-discipline
- Reference traceability-discipline/SKILL.md as format exemplar

## Authoring methodology

1. Propose 3-5 structural variants
2. Draft best 2-3 as complete SKILL.md files
3. Test each with claude -p using a NOVEL source: normal + adversarial
4. Compare, recommend, include a hybrid if top two complement each other
5. I pick or we iterate. Deploy when solid.

## In-session verification

Use --project-dir for workspace paths:

    prompt-runner run <prompt-file> --project-dir <workspace>

NEVER run two methodology-runner instances on the same workspace.

# project

This directory is the canonical promotion target for real implementation
output.

Use it for the built project itself:

- `project/src/`
  - real implementation code
- `project/tests/`
  - tests for the promoted implementation
- `project/docs/`
  - project-specific documentation
- `project/scripts/`
  - project-local helper scripts

This is distinct from:

- `tools/`
  - the reusable tooling products
- `sample/`
  - reference example bundles
- `work/`
  - transient execution workspaces
- root `docs/`
  - repository-level documentation

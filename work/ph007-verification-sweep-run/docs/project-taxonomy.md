# Project Taxonomy

This fixture workspace is a minimal project used by prompt-runner phase tests.

## Categories

### docs/requirements/
- Purpose: canonical requirements artifacts for the fixture project.

### docs/features/
- Purpose: feature-specification artifacts derived from requirements.

### docs/architecture/
- Purpose: architecture artifacts such as stack manifests.

### docs/design/
- Purpose: solution-design and interface-contract artifacts.

### docs/simulations/
- Purpose: simulation definitions and related verification-planning artifacts.

### docs/implementation/
- Purpose: implementation-plan artifacts.

### docs/verification/
- Purpose: verification-report artifacts that define end-to-end verification plans and coverage traceability.

### scripts/
- Purpose: deterministic validation helpers executed during prompt validation.

### (project root)
- Purpose: fixed project-level files required by tooling or agent instructions.
